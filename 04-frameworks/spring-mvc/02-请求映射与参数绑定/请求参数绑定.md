# 请求参数绑定

我们在用表单提交数据时，默认情况下，数据是以 key=value 的形式提交给后台程序的。通过 SpringMVC 的请求参数绑定功能，我们可以很轻松地把提交的数据绑定到控制器方法的参数上，前提是请求参数名和方法参数名必须一致。

假如我们要查询 id 为10的用户信息，表现层的代码可能像这样：

<a href="account/findAccount?id=10">查询用户</a>

控制器代码像这样就可以获取传过来的 id：

```java
@Controller
@RequestMapping("/account")
public class AccountController {
    @RequestMapping("/findAccount")
    public String findAccount(Integer id) {
        System.out.println(id); // 10
        return "success";
    }
}
```

请求参数绑定不仅支持基本类型和 String 类型，还支持 POJO 类型参数（实体类以及关联的实体类）、数组和集合类型参数（包括 List、Map）。

参数绑定的过程是自动的，但是必须遵循使用要求：

如果是基本类型或者 String 类型：要求表单中参数名必须与控制器中方法的形参名保持一致，注意区分大小写。

如果是 POJO 类型，或者它的关联对象：要求表单中参数名和 POJO 类的属性名保持一致，并且控制器方法的参数类型是 POJO 类型。

如果是集合类型，有两种方式：①要求集合类型的请求参数必须在 POJO 中。在表单中请求参数名称要和 POJO 中集合属性名称相同。给 List 集合中的元素赋值，使用下标。给 Map 集合中的元素赋值，使用键值对。②接收的请求参数是 json 格式数据。需要借助一个注解实现。

还可以实现一些数据类型自动转换。内置转换器全都在：org.springframework.core.convert.support 包下。

如果用户提交的参数不能转换为方法中指定的参数类型，此时我们可能需要用到自定义类型转换器。
