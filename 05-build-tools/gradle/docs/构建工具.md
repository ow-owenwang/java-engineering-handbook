# 构建工具

Gradle、Maven 和 Groovy 经常一起出现，但它们的本质是完全不同的三类工具或语言：

## 🎯 详细解释

### 🛠️ Gradle：现代构建系统（构建工具）

作用： 管理项目的构建流程（编译、测试、打包、依赖管理等）

支持语言： Java、Kotlin、Groovy、Scala 等

DSL： 默认使用 Groovy DSL（也支持 Kotlin DSL）

特点：

任务可编程：可以写逻辑判断、循环、条件构建

插件化系统强大（如 Android Gradle Plugin）

比 Maven 更快（增量构建、守护进程、缓存）

🔧 典型文件（Groovy DSL）：

plugins {    id 'java'}repositories {    mavenCentral()}dependencies {    implementation 'org.apache.commons:commons-lang3:3.12.0'}

安装本地 Gradle + IDEA 配置

1. 进入官网下载或国内镜像：

• 清华镜像 Gradle

• Gradle 官网

2. 解压后设置环境变量（可选）：

export GRADLE_HOME=/path/to/gradle-8.5export PATH=$GRADLE_HOME/bin:$PATH

3. IDEA 设置使用本地 Gradle：

Preferences / Settings → Build, Execution, Deployment → Build Tools → Gradle

• Gradle 配置方式 → 选择 “Use local Gradle distribution”

• 指定目录，如 /Users/yourname/gradle-8.5

### ⚙️ Maven：传统构建工具（构建工具）

作用： 管理项目构建、生命周期和依赖（与 Gradle 同类）

配置语言： XML（不可编程）

理念： “约定优于配置” —— 项目结构和流程是固定的

特点：

学习门槛低，企业广泛使用

插件体系成熟但不如 Gradle 灵活

配置较繁琐（XML 写法冗长）

🔧 典型文件（pom.xml）：

<project>  <modelVersion>4.0.0</modelVersion>  <groupId>com.example</groupId>  <artifactId>myapp</artifactId>  <version>1.0.0</version>  <dependencies>    <dependency>      <groupId>org.apache.commons</groupId>      <artifactId>commons-lang3</artifactId>      <version>3.12.0</version>    </dependency>  </dependencies></project>

### 📝 Groovy：一种脚本语言（编程语言）

作用： 是 Gradle 默认构建脚本使用的语言

语法特点：

基于 Java，可与 Java 无缝互操作

动态语言，语法简洁、支持 DSL 编程

用法场景：

编写 Gradle 构建脚本（Groovy DSL）

编写 Jenkins 脚本（如 pipeline）

编写动态脚本或工具类（例如小型自动化脚本）

Groovy 本身不是构建工具，但它是 Gradle 使用的脚本语言（就像你可以用 Bash 写 Makefile 的构建脚本）。

## ✅ 开发建议

新项目（Kotlin / Java）：推荐使用 Gradle + Kotlin DSL

企业传统项目：很多还在用 Maven

学习脚本语言/Gradle DSL：可以学点 Groovy 基础
