# 国际化

在信息技术领域，国际化与本地化（英文：internationalization and localization）是指修改软件使之能适应目标市场的语言、地区差异以及技术需要。

国际化是指在设计软件，将软件与特定语言及地区脱钩的过程。当软件被移植到不同的语言及地区时，软件本身不用做内部工程上的改变或修正。本地化则是指当移植软件时，加上与特定区域设置有关的信息和翻译文件的过程。

国际化和本地化之间的区别虽然微妙，但却很重要。国际化意味着产品有适用于任何地方的“潜力”；本地化则是为了更适合于“特定”地方的使用，而另外增添的特色。用一项产品来说，国际化只需做一次，但本地化则要针对不同的区域各做一次。这两者之间是互补的，并且两者合起来才能让一个系统适用于各地。

基于他们的英文单字长度过长，常被分别简称成i18n（18意味着在“internationalization”这个单字中，i和n之间有18个字母）。

国际化要做的工作包括处理日期格式、度量衡、语言本身和书写方向等在不同国家之间的差异。

整个过程也不复杂，首先需要定义含有不同语言描述的文件，然后根据不同国家地域读取不同的文件进行展示。

国际化开发常用类有：

Locale

ResourceBundle

MessageFormat

ResourceBundle

JDK 中的 ResourceBundle 读取资源文件使用的默认编码方式（ISO-8859-1）读取文件。

解决方案：

将中文字符转为 Unicode 字符，这样属性文件里就没有中文了。

String val = new String(rb.getString(key).getBytes(StandardCharsets.ISO_8859_1), StandardCharsets.UTF_8);，.properties 文件需要指定为 UTF-8 编码。

以上两种方法推荐使用第一种，中文转换为 Unicode 可以使用命令：

native2ascii zh_CN.properties lang_zh_CN.properties

也可以通过 IDEA 自动转化。

MessageFormat

这个类的作用是用来替换占位符的，类似于 String.format(String pattern,Object...args)。
