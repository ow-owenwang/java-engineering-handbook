# 面向对象

## 概述

面向对象是一种开发思想，

类是同一类事物的总称，例如鸟类、家禽类。鸟类包括很多种鸟，但是他们都具有很多的相似点，比如有翅膀、会飞。

类实质上就是封装对象属性和行为的载体，而对象则是类抽象出来的一个实例。

开发语言在不断发展，从机器语言，到汇编，到高级语言，再到第四代语言。软件开发方法也在不断发展，从面向过程，面向对象，到面向切面等。虽然这些都在不断发展，但其所追求的目标却一直没变，这些目标就是：

降低软件开发的复杂度

提高软件开发的效率

提高软件质量：可维护性，可扩展性，可重用性等。

### 封装

封装是面向对象编程的核心思想。将对象的属性和行为封装起来，从而达到隐藏其实现细节的目的。例如，用户使用计算机时，只需要使用手指敲击键盘就可以实现一些功能，无须知道计算机内部是如何工作的。它保证了类内部数据结构的完整性，使用该类的用户只能通过暴露的接口操作特定的数据。这样就避免了外部操作对内部数据的影响，从而提高了程序的可维护性。

### 局部变量

方法内的变量就是局部变量，包括形参。局部变量在方法被执行时创建，在方法执行结束时被销毁。

#### 有效范围

局部变量的有效范围也可以称为变量的作用域。

在嵌套作用域中不可以定义相同名称和类型的局部变量。

### 构造方法

构造方法是一个与类同名的方法，对象的创建就是通过构造方法完成的。每当实例化一个类的对象时，都会自动调用构造方法。

构造方法没有返回值，但是不用 void 修饰。

一般在构造方法中给成员变量进行赋值，这样实例化一个对象时，相应的成员变量也将被初始化。

如果类中没有明确定义构造方法，编译器会自动创建一个不带参数的默认构造方法。如果含有的构造方法都是有参的，那么编译器不会为该类创建一个默认无参的构造方法，这种情况下试图调用无参构造方法实例化一个对象时，编译器会报错。

### 主方法

主方法是类的入口点，它定义了程序从何处开始，Java 编译器通过主方法执行程序。

特点：

主方法是静态的，如要直接在主方法中调用其他方法，则被调用的方法必须也是静态的。

主方法没有返回值。

## 对象

### 对象的创建

对象可以认为是在一类事物中抽象出的某一个特例，可以通过这个特例来处理这类事物出现的问题。在 Java 语言中通过 new 关键字来创建对象。

Person p = new Person();

每个对象都是独立的，在内存中占有独立的内存地址。

### 对象的销毁

每个对象都有生命周期，当对象的生命周期结束时，分配给该对象的内存将被回收。在 Java 中有一套完整的垃圾回收机制，用户不必担心废弃的对象占用内存，垃圾回收器将回收无用的但占用内存的资源。

垃圾回收的过程是程序自动完成的，不受程序员控制，但是可以通过把对象赋值为 null 来让虚拟机更快地回收该对象所占用的内存。
