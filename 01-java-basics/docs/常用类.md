# 常用类

## Object

java.lang.Object 类是 Java 语言中的根类，即所有类的父类。它中描述的所有方法子类都可以使用。在对象实例化的时候，最终找的父类就是 Object。

如果一个类没有特别指定父类，那么默认则继承自 Object 类。

public class MyClass /*extends Object*/ {}

Object 类上有几个方法是平时开发中经常用到的：

public String toString()：返回该对象的字符串表示。

public boolean equals(Object obj)：某个对象是否与此对象“相等”。

下面逐个进行介绍：

toString() 方法返回该对象的字符串表示，其实该字符串内容就是对象的类型+@+内存地址值。由于 toString() 方法返回的结果是内存地址，而在开发中，经常需要按照对象的属性得到相应的字符串表现形式，因此也需要重写它。

在使用类似 System.out.println() 的输出语句输出对象时，默认会调用对象的 toString() 方法。

调用成员方法 equals() 并指定参数为另一个对象，则可以判断这两个对象是否是相同的。这里的“相同”有默认和自定义两种方式。

如果没有覆盖重写 equals() 方法，那么 Object 类中默认进行 == 运算符的对象地址比较，只要不是同一个对象，结果必然为 false。

如果希望进行对象的内容比较，即所有或指定的部分成员变量相同就判定两个对象相同，则可以覆盖重写 equals() 方法。

## System

java.lang.System 类中提供了大量的静态方法，可以获取与系统相关的信息或系统级操作，在 System 类的 API 文档中，常用的方法有：

public static long currentTimeMillis()：返回当前时间与1970年1月1日0时0分0秒之间以毫秒为单位的时间差。一般用来测试程序运行效率。

public static void arraycopy(Object src, int srcPos, Object dest, int destPos, int length)：将数组中指定的数据拷贝到另一个数组中。

计算世界时间的主要标准有：

UTC(Universal Time Coordinated)

GMT(Greenwich Mean Time)

CST(Central Standard Time)

验证 for 循环打印数字1-9999所需要使用的时间（毫秒）：

long start = System.currentTimeMillis();for (int i = 1; i <= 9999; i++) {    System.out.println(i);}long end = System.currentTimeMillis();System.out.println("程序共耗时：" + (end - start) + "毫秒");

## Date

long getTime()： 把日期转换为对应时间的毫秒值

## Calendar

java.util.Calendar 是日历类。它在 Date 后出现，替换掉了许多 Date 的方法。该类将所有可能用到的时间信息封装为静态成员变量，方便获取。日历类就是方便获取各个时间属性的。

Calendar 类是一个抽象类，里边提供了很多操作日历字段的方法（YEAR、MONTH、DAY_OF_MONTH、HOUR）

由于语言敏感性，Calendar 类无法直接创建对象使用，而是通过静态方法创建，返回子类对象，里边有一个静态方法叫 getInstance()，该方法返回了 Calendar 类的子类对象。

Calendar 类中提供很多成员常量，代表给定的日历字段：

## Objects

在JDK7添加了一个 Objects 工具类，它提供了一些方法来操作对象，它由一些静态的实用方法组成，这些方法是null-save（空指针安全的）或null-tolerant（容忍空指针的），用于计算对象的 hashCode、返回对象的字符串表示形式、比较两个对象。

在比较两个对象的时候，Object 的 equals() 方法容易抛出空指针异常，而 Objects 类中的 equals() 方法就优化了这个问题。

public static boolean equals(Object a, Object b)：判断两个对象是否相等。源码如下：

public static boolean equals(Object a, Object b) {    return (a == b) || (a != null && a.equals(b));}

他有效地避免了 Object 对象的 equals() 方法抛出空指针异常。

## Random

此类的实例用于生成伪随机数。

public Random()：创建一个新的随机数生成器。

nextInt()：获取一个随机的int数字（范围是int所有范围，有正负两种）。

nextInt(3)：获取一个随机的int数字（参数代表了范围，左闭右开区间），实际上代表的含义是：[0,3)，也就是0~2。

## Scanner

一个可以解析基本类型和字符串的简单文本扫描器。

public Scanner(InputStream source) ：构造一个新的 Scanner ，它生成的值是从指定的输入流扫描的。System.in 系统输入指的是通过键盘录入数据。

nextInt()：获取键盘输入的一个 int 数字

next()：获取键盘输入的一个字符串

## StringBuilder

由于 String 类的对象内容不可改变，所以每当进行字符串拼接时，总是会在内存中创建一个新的 String 对象，既耗时，又浪费空间。为了解决这一问题，出现了 java.lang.StringBuilder 类。

StringBuilder 又称为可变字符序列，它是一个类似于 String 的字符串缓冲区，通过某些方法调用可以改变该序列的长度和内容。

它的内部拥有一个数组用来存放字符串内容，进行字符串拼接时，直接在数组中加入新内容。StringBuilder 会自动维护数组的扩容。默认16字符空间，超过自动扩充。

StringBuilder 已经覆盖重写了 Object 当中的 toString 方法，通过 toString 方法，StringBuilder 对象将会内容转换为不可变的 String 对象。

StringBuilder 可以存储任意类型的数据。

StringBuilder()：构造一个不带任何字符的字符串生成器，其初始容量为 16 个字符。

StringBuilder(String str)：构造一个字符串生成器，并初始化为指定的字符串内容。

public StringBuilder append()：添加任意类型数据的字符串形式，并返回当前对象自身。

StringBuilder 和 String 可以互相转换：

StringBuilder(String str)：构造一个字符串生成器，并初始化为指定的字符串内容。

public String toString()：将当前 StringBuilder 对象转换为 String 对象。

## DateFormat

java.text.DateFormat 是日期/时间格式化子类的抽象类，通过这个类可以完成日期和文本之间的转换，也就是可以在 Date 对象与String 对象之间进行来回转换。

DateFormat 类是一个抽象类，无法直接创建对象使用，可以使用 DateFormat 类的子类 java.text.SimpleDateFormat。这个类需要一个模式（格式）来指定格式化或解析的标准。

常用的格式规则为：

标识字母（区分大小写）

含义

y

年

M

月

d

日

H

时

m

分

s

秒

SimpleDateFormat(String pattern)：用给定的模式和默认语言环境的日期格式符号构造 SimpleDateFormat。

String format(Date date)：按照指定的模式，把 Date 日期，格式化为符合模式的字符串。

Date parse(String source) ：把符合模式的字符串，解析为 Date 日期。

## Math

java.lang.Math 类包含用于执行基本数学运算的方法，如初等指数、对数、平方根和三角函数。类似这样的工具类，其所有方法均为静态方法，并且不会创建对象，调用起来非常简单。

## Arrays

java.util.Arrays 此类包含用来操作数组的各种方法，比如排序和搜索等。其所有方法均为静态方法，调用起来非常简单。

## Date

java.util.Date：表示日期和时间的类。表示特定的瞬间，精确到毫秒。

毫秒值的作用：可以对时间和日期进行计算。如：当要计算 2099-01-03 到 2088-01-01 中间一共有多少天，可以把日期转换为毫秒进行计算，计算完毕，在把毫秒转换为日期。

时间原点(0毫秒)：1970 年 1 月 1 日 00:00:00(英国格林威治)。而中国属于东八区，会把时间增加8个小时，也就是 1970年1月1日 08:00:00
