## 标识符

Java 对变量、方法和类等要素命名时使用的字符序列称之为标识符（凡是自己可以起名字的地方都叫标识符）。

标识符定义规则：

由任意顺序的字母、下划线、美元符号和数字组成。

数字不可以开头。

不可以使用关键字和保留字，但能包含关键字和保留字。

Java 中严格区分大小写，长度无限制。

标识符不能包含空格。

Java 语言使用 Unicode 标准字符集，所以还可以使用中文等其他文字作为标识符。但是为了提高可读性，尽量做到见名知意。

字符类型用于存储单个字符，在定义字符类型变量时，要以单引号包裹。

转义字符 具有特定的含义，不同于字符原有的意义。将转义字符赋值给字符变量时，与字符常量值一样需要使用单引号。

## 变量与常量

### 变量

变量是内存中的一个存储区域，该区域有自己的名称（变量名）和类型（数据类型）。变量是通过使用变量名来访问这块区域存储的数据的，该区域的数据可以在同一类型范围内不断变化。变量名必须是合法的标识符。指定数据类型是为了让编译器知道分配多少内存空间。

int age = 30;
char c = 'r';

变量可以根据声明位置的不同将变量分为成员变量和局部变量，根据数据类型的不同分为基本数据类型变量和引用数据类型变量。

基本数据类型：

数值型（整数类型和浮点类型）

字符型

布尔型

引用数据类型：

类

接口

数组

成员变量：

在方法体外，类体内声明的变量称为成员变量。成员变量中又可分为实例变量和静态变量。

实例变量：不以 static 修饰，可以通过类的实例来访问。

类变量（静态变量）：以 static 修饰，可以通过类名直接访问。

public class Demo {
    static int i1 = 1000; //静态变量
    int i2 = 200; //实例变量

    public static void main(String[] args) {
        System.out.println(Demo.i1); //通过类直接访问

        Demo d = new Demo();
        System.out.println(d.i2);
    }
}

局部变量：

在方法体内部声明的变量称为局部变量。局部变量可与成员变量的名字相同，此时成员变量将被屏蔽。局部变量可细分为形参、方法局部变量和代码块局部变量。

形参：方法签名中定义的变量。

方法局部变量：在方法内定义的变量

代码块局部变量：在代码块内定义的变量。

局部变量除形参外，需显式初始化。

局部变量的生命周期取决于方法，当方法被调用时，Java 虚拟机为方法中的局部变量分配内存空间，当该方法的调用结束后，则会释放方法中局部变量占用的内存空间，局部变量也将会销毁。

### 常量

变量的值是可以变化的，而常量是在整个程序运行过程中一直不会改变的，只能被赋值一次。常量一般用来为所有的对象共享数据。常量名通常使用大写字母，但不是必须的。当定义的常量是成员变量时，必须定义时就初始化它的值，否则会产生编译错误。

public class FinalDemo {
    static final double PI = 3.14;

    public static void main(String[] args) {
        final int age;
        age = 30;
        System.out.println(PI);
        System.out.println(age);
    }
}

对于除号，整数除和小数除是有区别的，整数之间做除法时，只保留整数部分而舍弃小数部分。

加号运算符对于数值来说，就是加法。然而对于字符 char 类型来说，在计算之前，char 会被提升成为 int，然后再计算。

对于字符串来说，加号代表字符串连接操作。任何数据类型和字符串进行连接的时候，结果都会变成字符串。

在进行除法运算时，0不能作为除数。

### 自增和自减运算符

自增、自减运算符是单目运算符，可以放在操作元之前，也可以放在操作元之后。操作元必须是一个整型或浮点型变量。它们的作用使变量的值增1或减1。放在操作元前面的自增、自减运算符，会先将变量的值加1或减1，然后再参与表达式的运算。放在操作元后面的自增、自减运算符，会先使变量参与表达式的运算，然后再将该变量加1或减1。而当前++或者后++作为一个独立的语句（前--和后--一样），不与其他操作运算一起使用时，对运算的结果没有影响。

int x = 2;
int y = ++x;
System.out.println(x); //3
System.out.println(y); //3

y = x++;
System.out.println(x); //4
System.out.println(y); //3

“&”和“&&”的区别（“|”和“||”的区别同理）：

### 三元运算符

一元运算符： 只需要一个数据就可以进行操作的运算符。例如：取反!、自增++、自减--。

二元运算符： 需要两个数据才可以进行操作的运算符。例如：加法+、赋值=。

三元运算符： 需要三个数据才可以进行操作的运算符。

格式：条件表达式 ? 值1 : 值2

运算规则：若条件表达式的值为 true，整个表达式取值1，否则取值2。其中值1和值2可以是某种数据类型的值，也可以是表达式。

int a = 10;
int b = 20;
System.out.println(a > b ? "hello" : 10000);

String result = a > b ? "a 大" : "b 大"; // 用变量接收运算结果时，两个值必须为相同数据类型。
System.out.println(result);

三元运算符在一定程度上可以与 if-else 互换，可简化 if-else 语句，三元运算符一定可以转换为 if-else ，反之则不一定。

## 基本数据类型转换

Java 程序中要求参与计算的数据，必须要保证数据类型的一致性，如果数据类型不一致将发生类型的转换。类型转换是将一个值从一种类型更改为另一种类型的过程。

注意：

可以将任意类型的数据转换为 String 类型。

低精度数据类型向高精度数据类型转换时，永远不会溢出，并且总是成功。反之，必然会有信息的丢失，并且有可能失败。

### 隐式转换

隐式转换是从容量小（取值范围小）的类型向容量大（取值范围大）的数据类型转换，系统将自动执行（自动类型转换），程序员无须进行任何操作。

int x = 50;
float y = x;
System.out.println(y); //50.0

// byte 类型内存中占有一个字节，在和 int 类型运算时会提升为 int 类型，自动补充3个字节，因此计算后的结果还是 int 类型。
byte b = 3;
int sum = b + x;

数据类型按容量大小排序为：double > float > long > int > char、short、byte

当有多种类型的数据混合运算时，系统首先自动将所有数据转换成容量最大的那种数据类型，然后再进行计算。byte、short、char 之间不会相互转换，他们三者在计算时首先转换为 int 类型。

当把任何基本类型的值和字符串值进行连接运算时(+)，基本类型的值将自动转化为字符串类型。

将容量大的数据类型转换为容量小的数据类型时，必须加上强制转换符。当执行 显示类型转换 （或称之为 强制类型转换 ）时可能会导致精度损失，要格外注意。 boolean 类型不可以转换为其它的数据类型。字符串不能直接转换为基本类型，但通过基本类型对应的包装类则可以实现把字符串转换成基本类型。当把整数赋值给一个 byte、short、int、long 型变量时，不可以超出这些类型的取值范围，否则必须进行强制类型转换。

int i1 = (int) 45.234; //45
int i2 = (int) 'd'; //100
long l = (int) 456.7F; //456
byte b = (byte) 129; //-127

String s = "430";
int i3 = Integer.parseInt(s);

## 注释

在程序代码中适当地添加注释可以提高程序的可读性和可维护性。可读性第一，效率第二。程序中注释要占程序代码总量的20%~50%。

单行注释：

// 为单行注释标记，从符号 // 开始直到换行为止的所有内容均作为注释而被编译器忽略。

int age = 30; //年龄

多行注释：

/* */ 为多行注释标记，星号之间的所有内容均为注释内容。

int age = 30; /* 年龄 */

文档注释：

当文档注释出现在声明（如类的声明、类的成员变量的声明、类的成员方法声明等）之前时，会被 javadoc 文档工具读取作为 javadoc 文档的内容。

## 编码规范

在开发的过程中要养成良好的编码规范，因为规整的代码格式会给程序的开发与日后的维护提供很大的便利。

每条语句要单独占一行，一条命令要以分号结束。

## 命名规范

类名/接口名：首字母大写，两个单词以上，每一个单词首字母大写。

属性/方法/变量：驼峰式命名，首字母小写，如果两个单词以上，第二个以后的每一个首字母大写。

构造方法：与类名一致，类中唯一的大写字母开头的方法。

静态常量：全部字母大写，通过 _ 做具体说明。

包名：全部字母小写，java关键字都是小写，注意与关键字不要冲突。
